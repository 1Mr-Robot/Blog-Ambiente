# Guía de Contribución

## Requisitos Previos

Para ejecutar este proyecto necesitas:

- Un navegador web moderno (Chrome, Firefox, Edge, Safari)
- Un editor de texto o IDE (VS Code, Sublime Text, Notepad++, etc.)

## Cómo Ejecutar el Proyecto

### Opción 1: Abrir directamente en el navegador

1. Navega al directorio del proyecto
2. Abre el archivo `pages/index.html` en tu navegador
3. También puedes hacer doble clic en el archivo HTML desde el explorador de archivos

### Opción 2: Usando un servidor local (recomendado)

#### Con VS Code (Extensión Live Server)

1. Instala la extensión "Live Server" en VS Code
2. Haz clic derecho en `pages/index.html`
3. Selecciona "Open with Live Server"

## Estructura del Proyecto

```
ProyectoAmbiente/
├── pages/          # Archivos HTML del blog
├── styles/         # Archivos CSS
├── images/         # Imágenes del proyecto
└── README.md       # Documentación del proyecto
```

## Cómo Contribuir

1. Haz un fork del repositorio
2. Crea una rama para tu característica (`git checkout -b feature/nueva-caracteristica`)
3. Realiza tus cambios ycommits
4. Envía un pull request

## Pautas de Estilo

- Mantén el código limpio y organizado
- Usa nombres descriptivos para archivos y clases
- Asegúrate de que los cambios sean responsivos
- Verifica que las imágenes tengan rutas correctas

## Problemas Comunes

- **Imágenes no cargan**: Verifica que las rutas en el HTML apunten correctamente a la carpeta `images/`
- **Estilos no se aplican**: Confirma que los archivos CSS estén correctamente vinculados en el HTML
- **Links rotos**: Asegúrate de que los hipervínculos apunten a archivos existentes
